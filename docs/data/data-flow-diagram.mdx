Also known as `DFD`

A DFD charts the oath of data as it flows outside the boundries of a system. 

## Pros
`-` Beneficial for communicating existing system knowledge to the users. <br />
`-` A straightforward graphical technique which is easy to recognise. <br />
`-` DFD's can provide a detailed experience representation of system components. <br />
`-` Provides detailed information about the system flow

## Cons
`-` Takes a long time to create <br />
`-` Physical considerations are left out
